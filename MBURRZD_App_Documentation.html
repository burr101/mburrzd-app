Introduction
Executive Overview

The MBURRZD mobile app is a fully custom iOS shopping experience built for the MBURRZD luxury big-and-tall menswear brand. The app connects directly and in real time to the existing Shopify store, meaning all products, prices, inventory levels, and collections update automatically — no manual sync required.

The aesthetic is inspired by the world's most considered luxury fashion houses — Loro Piana, Fear of God, Amiri — translated into a mobile experience that matches the brand's premium positioning. Every screen has been built from scratch to reflect the MBURRZD identity: deep black, warm gold, Georgia serif typography.

Design Principles

Visual Identity

Black (#0A0A0A) backgrounds, warm gold (#C9A96E) accents, Georgia serif for editorial headings, Inter for UI text.

Customer Experience

Curated, editorial home screen. Premium product photography at full bleed. Minimal friction from browse to checkout.

Commerce

100% live Shopify data. Cart and checkout handled natively by Shopify — secure, trusted, PCI compliant.

Personalisation

Find Your Fit quiz tailors size recommendations to height, weight, and body type — a differentiator for big & tall.

Product
App Features & Screens

The app is organised into five main tabs accessible from the bottom navigation bar. Each tab contains one or more screens.

Home

Hero Section

Full-bleed editorial hero image with animated gradient overlay, seasonal copy, and a direct CTA to the shop. Logo displayed at top.

Brand Positioning

Three animated lifestyle copy blocks fade in on scroll, separated by gold hairline dividers. Sets editorial tone.

Collections & New Arrivals

Horizontally scrollable collection tiles. Vertically scrollable new arrivals grid. Both pull live from Shopify.

Worn By

Editorial influencer/brand social proof section. Currently features @mburrzd_ with real product photography.

Testimonials

Auto-rotating customer quote carousel. Fades between three testimonials every 4.5 seconds.

Fit Concierge Card

Gold-bordered call-to-action card directing customers to the Find Your Fit quiz.

Shop

A full product grid with real-time Shopify inventory. Customers can search by keyword, sort by price or newest, and filter by collection. Each product card shows the product image, name, a feature descriptor line, and price. Sold-out products are clearly labelled.

Product Detail

Full product page with swipeable image gallery, colour swatch selector (extracted from Shopify variants), size selector with active state, expandable description accordions, "CRAFTED FOR PRECISION" key features section, and trust badges. The Add to Bag button is prominently placed in gold.

Find Your Fit

A four-step guided quiz unique to the MBURRZD experience. The customer selects their height, weight range, body type, and fit preference. The app then computes and displays a recommended size (S through 3XLT) with chest and waist measurements and personalised fit notes.

Bag & Checkout

Cart screen shows all added items with quantity controls and live price totals. Checkout opens the secure Shopify checkout in a web view — meaning payment processing is handled entirely by Shopify (PCI DSS compliant). No card data ever passes through the app code.

Account

Unauthenticated State

  • · Sign In with Apple (iOS-native, one tap)
  • · Email / password sign in
  • · New account registration
  • · Member perks showcase
  • · Privacy Policy link

Authenticated State

  • · Avatar with gold border + member badge
  • · Saved fit profile card
  • · Quick links: Wishlist, Orders, Fit Quiz
  • · Privacy Policy & Settings
  • · Sign Out + Delete Account

Getting Started
Development Setup

The following steps prepare a Windows machine to build, run, and edit the app. This only needs to be done once per computer.

Prerequisites

  • 1
    Install Node.js

    Download the LTS version from nodejs.org and run the installer with all defaults. Provides the JavaScript runtime used by the build tools.

  • 2
    Install Git

    Download from git-scm.com/download/win — accept all defaults. Git tracks every change to the code and enables collaboration and backup.

  • 3
    Install the EAS CLI

    Open PowerShell and run:

    npm install -g eas-cli

    EAS (Expo Application Services) is the cloud build service that compiles the app.

  • 4
    Navigate to the project and install dependencies
    cd "C:\Users\Royston_Nnamdi\OneDrive - Dell Technologies\Documents\Windsurf_Projects\AppBuild" npm install
  • 5
    Log in to EAS
    $env:NODE_TLS_REJECT_UNAUTHORIZED=0; eas login

    The NODE_TLS_REJECT_UNAUTHORIZED=0 prefix is required on Dell corporate laptops due to the company SSL proxy.

Local Preview (No Build Required)

Install Expo Go from the App Store on your iPhone, then run npm start in the project folder. Scan the QR code that appears in PowerShell. The app will open instantly. Note: Sign In with Apple and some native features require a real build.

Building & Deploying

Build Profiles

ProfilePurposeOutputHow to Install

Running a Preview Build (Test Devices)

  • 1
    Run the build command
    $env:NODE_TLS_REJECT_UNAUTHORIZED=0; eas build --platform ios --profile preview
  • 2
    Authenticate with Apple when prompted

    Enter your Apple ID and password. EAS will log in, sync capabilities, and generate a new provisioning profile if needed.

  • 3
    Select your devices

    Use the spacebar to select both iPhone and iPad from the list, then press Enter.

  • 4
    Wait for the build to complete

    Free tier builds take approximately 15–30 minutes. Progress can be tracked at expo.dev.

  • 5
    Install via QR code

    When the build finishes, a QR code appears in the PowerShell terminal. Scan it with the iPhone camera to install. Always use the terminal QR code — the expo.dev web link is unreliable on these devices.

Running a Production Build (App Store)

$env:NODE_TLS_REJECT_UNAUTHORIZED=0; eas build --platform ios --profile production

This generates a distribution-signed .ipa for App Store submission. Download the file from expo.dev when complete and upload it using the Transporter app (Mac App Store, free).

Important

Always run a preview build and test on device before running a production build. The production process requires full Apple credentials and generates a signed binary that counts toward your App Store version history.

Version Control
Saving to GitHub

GitHub stores the complete history of every change made to the app. Think of it as a backup system that never forgets — you can restore any previous version of the app at any time. It also enables collaboration if you bring in additional developers.

First-Time Setup

  • 1
    Create a GitHub account

    Go to github.com and sign up. Choose a username that represents you or the brand.

  • 2
    Create a new private repository

    Click the + icon → New repository → name it mburrzd-app → set to Private → click Create repository.

  • 3
    Create a .gitignore file

    In the AppBuild folder, create a plain text file called .gitignore with this content to exclude large/generated folders:

    node_modules/ .expo/ dist/ android/ ios/ *.orig.* web-build/ .DS_Store
  • 4
    Push the code to GitHub
    cd "C:\Users\Royston_Nnamdi\OneDrive - Dell Technologies\Documents\Windsurf_Projects\AppBuild" git init git config user.email "rstonokafor@gmail.com" git config user.name "Royston" git add . git commit -m "Initial commit: full MBURRZD app v1.0" git remote add origin https://github.com/YOUR-USERNAME/mburrzd-app.git git push -u origin main

    When asked for a password, use a Personal Access Token from GitHub (Settings → Developer settings → Personal access tokens → Generate new token → tick "repo").

Saving Future Changes

Any time you update the app, save your changes to GitHub with three commands:

git add . git commit -m "Brief description of what you changed" git push

Getting the Code on a New Computer

git clone https://github.com/YOUR-USERNAME/mburrzd-app.git cd mburrzd-app npm install

App Store Submission

Pre-Submission Checklist

  • Apple Developer Program membership active ($99/year at developer.apple.com)
  • Supabase: Email provider enabled (Authentication → Providers → Email)
  • Supabase: Apple provider enabled with Team ID A32PJN4364
  • Privacy Policy hosted at a public URL (e.g. mburrzd.com/privacy)
  • App icon is PNG format, 1024×1024px, no transparency
  • App Store screenshots prepared (6.7" iPhone minimum required)
  • Account deletion flow tested and working
  • Sign In with Apple tested on a real device
  • Production build created and downloaded

Required Screenshots

DeviceResolutionRequired?
iPhone 6.7" (15 Pro Max)1290 × 2796 pxYes — mandatory
iPhone 6.5" (14 Plus)1284 × 2778 pxRecommended
iPad 12.9"2048 × 2732 pxRequired (supportsTablet: true)

Submission Steps

  • 1
    Run the production build
    $env:NODE_TLS_REJECT_UNAUTHORIZED=0; eas build --platform ios --profile production
  • 2
    Download the .ipa from expo.dev

    Navigate to expo.dev → your project → builds → download the production .ipa file.

  • 3
    Upload with Transporter (Mac required)

    Open Transporter (free on Mac App Store) → sign in with Apple ID → drag in the .ipa → click Deliver.

  • 4
    Complete the App Store Connect listing

    Go to appstoreconnect.apple.com → My Apps → New App → fill in name (MBURRZD), bundle ID (com.mburrzd.fashion), category (Shopping), description, screenshots, privacy policy URL, and age rating.

  • 5
    Submit for review

    Click Submit for Review. Apple's review takes 24–72 hours for first submissions. You'll receive an email when approved or if changes are requested.

Tip — Use TestFlight First

Consider releasing via TestFlight (Apple's beta platform) before your full App Store submission. TestFlight builds are faster to approve (no review) and let you gather feedback from up to 10,000 external testers. Enable it in App Store Connect under the TestFlight tab.
